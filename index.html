HTML

HTML stands for Hypertext Markup Language, and is made up of text content (what you see), markup (what it looks like), references to other docs, and links to other pages. Note: plain text is plain text in HTML. You want to utilize "markup" to make it look different. (CSS)

HTML Markup

Markups include tags (opening/closing), which encompasses an entire "element" that is used to make the text behave a certain way. Utlizing scratchpad.io as a testing medium for code. Note: can "nest" tags. Note: can force multiple lines and paragraphs. Inline vs. Block.

Instructor Take-a-ways

What to take away from this lesson.The five most important things to take from this lesson are the following:

1 - What a Web Page Is
A web page is a text document written in a language called HTML. Web browsers read these documents, and then interpret and display them.

2 - How Coding Works
Coding happens when programmers write text in a language that a computer can understand. The computer can then follow the instructions the programmer wrote. For example, the computer might do this by making text like this:<br>

3 - Computers are Stupid
Programmers need to write exactly the way a computer understands (also known as writing with correct "syntax"). For example, if you forget to close a <b> tag, the computer won't be able to figure out what you had intended to make bold. This "stupidity" can be very frustrating, but it also gives programmers incredible power: if you know how to talk to a computer than you can tell it to do anything you want.

4 - Programmers Can't Remember Everything
There are too many details to keep everything in your head. And that's okay. If you forget how to make text italic in HTML, you can always just look it up.

5 - Basic HTML Vocabulary
You will be using HTML in the next few lessons, so it will be helpful if you're comfortable with the jargon.

* Tag: An HTML tag is always contained within angled brackets. Most tags have an opening tag (<p> for example) and a closing tag, (</p>). Some tags (called "void" tags) do not require a closing tag (like the <br> tag).

* Element: An HTML element refers to everything within a set of opening and closing tags.

* Attribute: This is a property of an HTML element. For example, to set thehref attribute of an anchor tag to the Udacity URL, you would write <a href="www.udacity.com">.

Lesson 2: Creating a Structured Document with HTML

HTML, CSS, JAVASCRIPT

There are three elements to a webpage: HTML, CSS, JAVASCRIPT. HTML is like the structure of a house. CSS is the style (i.e. color of walls, cabinetry, etc.). JAVASCRIPT is interactive, change an element (i.e. a chair, a lamp). Note: Developer Tools Tab in Google Chrome.

HTML and CSS are both languages (syntax+rules)(tag is word. HTML controls the structure of a web page. CSS controls the "style" of a page (how it looks). When programmers talk about DOM (Document Object Model) they are talking about the tree-like structure of a page (DOM Tree).

Lesson 3: Adding CSS Style to HTML Structure

CSS

CSS (Cascading Style Sheets) controls the “style” of a page, and is to prevent repeating. Example: “Style” - .description{color:red;} - turns words red inside of “description” box. Note: can do this in scratchpad.io by using <style> </style>.

Analogy: sheets are like laws. National Laws < State Laws < Municipality Laws < House Rules. CSS says House Rules become National Law. “Most specific rule applied” is the definition of Cascading. Font-style: normal, italic, oblique. (use CSS reference site to look up). Note: For “Code Comments,” HTML begins with . Everything in between is ignored. CSS begins with “/*" and ends with “*/".

Why Abstraction?

Inheritance requires abstract thinking. Abstraction is to avoid repetitive work. Inheritance = faster-loading, less bandwidth. Rules are interpreted in-between the brackets, and are applied to the specified element (i.e. h1). There is always a property and a value (property:value = declaration).

Box Sizing
  1. HTML elements are boxes and each box has 4 components.
  2. Because there are so many components to each box, it can often be hard to get the size of a box just right.
  3. There are two techniques you can use to help deal with sizing issues. First, set sizes in terms of percents rather than pixels. Second, set the “box-sizing” attribute to “border-box” for every element.
Box Positioning
  1. Divs are block elements (as opposed to inline), so by default they take up the entire width of a page.
  2. Adding the rule “display: flex;” to the appropriate CSS will override this behavior and let divs appear next to each other.
